# source_chapters 标注标准

`source_chapters` 列出编写该条目时需要参考的源材料章节，用于精准复读——只读这些章节的行号范围，不重读全文。

## 必须标注的场景

### 角色相关

1. **外貌变化**：角色外貌在不同章节有变化（如换装、受伤、成长）
2. **性格发展**：角色性格在不同阶段有明显变化
3. **关系转折**：角色关系在特定章节发生关键变化
4. **能力首次使用**：角色能力首次出现或首次展示具体效果
5. **背景揭示**：角色背景在特定章节揭示重要信息

### 世界观相关

6. **设定补充**：世界观设定在不同章节有补充或修改
7. **规则变化**：世界观规则在特定章节发生变化或例外
8. **历史事件**：历史事件在不同章节有详细描述

### 物品/能力相关

9. **物品首次出现**：重要物品首次出现并详细描述
10. **能力详细说明**：能力在特定章节有详细使用说明

### 剧情相关

11. **关键转折**：剧情在特定章节发生关键转折
12. **伏笔埋设**：伏笔在特定章节埋设，在后续章节揭示
13. **名场面**：重要场景、经典场面

## 标注格式

```yaml
entries:
  - name: 苏云_基础信息
    type: 角色
    part: basic
    path: 世界书/角色/苏云/基础信息.yaml
    source_chapters:
      - chapter: 第1章
        reason: "首次出场，基础外貌描写"
      - chapter: 第5章
        reason: "详细外貌描写，性格首次展现"
      - chapter: 第12章
        reason: "关系转折点，性格变化"
    notes: "编写时需综合三个章节的信息"
```

## 标注原则

- **必要性原则**：只标注确实需要参考的章节，不要过度标注
- **具体性原则**：说明每个章节的具体用途
- **完整性原则**：确保标注覆盖所有必要信息
