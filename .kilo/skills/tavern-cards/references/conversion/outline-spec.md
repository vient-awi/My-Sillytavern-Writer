# 大纲规范

大纲构建的信息分类、记录原则和格式规范。

## 大纲结构

大纲分为两部分：
- **chapters**：章节概要，用于理解故事结构和进度
- **全局信息**：世界观、区域、势力、角色、物品、重要章节、文风等，用于后续条目创作

```yaml
chapters:
  - name: 1-魔女小姐
    notes: 江酒向魔女小姐提分手，被灌下魔女秘药变成见习魔女
  - name: 2-天下无敌
    notes: 江酒已变成魔女女仆，莉莉丝离开去参加魔女茶会

worldview:
  - 科技光辉下不应有神秘力量，但存在魔女
  - 魔女学派：塑能、变化、召唤、启示

regions:
  - name: 下城区
    description: 上城区的镜像倒影，异类聚居的混乱地带

factions:
  - name: 静谧机关
    summary: 管理超凡事件的官方部门
    territory: 上城区
    key_members: [潘多拉（静谧魔女）, 桑落（执行官）]

characters:
  - name: 江酒
    birth_year: 1998           # 可选；年龄随时间变化的长篇用 birth_year，短篇可直接写 age
    gender: 男
    appearance:                # 可选；只记录源材料中明确提到的、偏离默认认知的外貌特征
      - 变成魔女后黑发黑眸
      - 肤色苍白，病态纤细
      - 左眼下有泪痣
      - 穿女仆装（黑色长裙+白色围裙+白色吊带袜+系带小皮鞋）
    identity: 渣男，有两位数前女友
    personality: 适应良好，脸皮厚
    relationship: 莉莉丝的魔女女仆
    quotes:
      - chapter: 1-魔女小姐
        context: 江酒坦白自己来赴约是为了分手
        text: 没错，今晚我是来跟你提分手的。
  - name: 莉莉丝
    gender: 女
    appearance:
      - 总是穿黑色长裙
      - 长发如墨，纯黑眼眸
      - 不涂口红，唇色苍白
    identity: 大魔女
    personality: 神秘优雅，与时俱进
    relationship: 江酒的主人
    quotes:
      - chapter: 2-天下无敌
        context: 莉莉丝要求江酒承认主仆关系
        text: 你不该叫我莉莉丝的，你应该叫我什么？

items:
  - name: 莉莉丝的魔杖
    summary: 六又三分之一寸长，接骨木材质
  - name: 魔女秘药
    summary: 喝下将成为见习魔女

important_chapters:
  - chapter: 1-魔女小姐
    reason: 魔女身份首次揭示，核心世界观设定
    quotes:
      - context: 莉莉丝宣告魔药效果，奠定主线身份转变
        function: 世界观揭示
        text: 喝下这瓶魔药，你将成为见习魔女。
  - chapter: 9-凡人之死
    reason: 年轻男人的悲惨生活，情感转折点

style_hints:                   # 可选；从源材料推断的文风信息，供需求对齐时参考
  perspective: 第三人称
  tone: 口语化、轻松
  mood: 轻松中带惊悚
```

## 记录规范

### chapters.notes

**必须记录**：每章都必须有 notes，概括本章核心内容。

**内容要求**：
- 主要事件或剧情转折
- 新角色出场或关系变化
- 重要设定揭示
- 字数：50-100 字

**示例**：

- 「江酒在酒吧向魔女小姐提分手，魔女小姐早有准备，揭示自己是真正的魔女，用魔法让酒吧所有人沉睡，召唤触手，给江酒灌下魔女秘药让他成为见习魔女作为惩罚」
- 「姜小白因邻居家半夜的奇怪声响失眠做噩梦，精神衰弱，雨中出门迷路误入莉莉丝酒吧，遇到女仆打扮的江酒」
- 「江酒进入年轻男人的记忆空间，目睹他悲惨的生活：底层工作、慢性肠胃炎、经济困顿，最终孤独病死，无人知晓」

### worldview

**记录内容**：
- 世界规则、物理法则、社会结构
- 历史背景、重大事件
- 让 AI 理解世界观的关键设定

**禁止**：
- 情节描述（属于 chapters.notes）
- 纯环境描写
- 通用常识

### factions

**记录内容**：
- 势力/组织名称
- 性质、目的（`summary`）
- 可选 `territory`：主要活动区域
- 可选 `key_members`：关键成员列表

### regions

**记录内容**：
- 区域/地点名称
- `scenes`：该区域下的具体场景列表
- 可选 `description`：区域概述

**要求**：
- 源材料中提到具体地点时记录，尤其是角色频繁活动的区域
- 区域粒度以源材料描述为准，不自行拆分或合并
- `scenes` 只列源材料中出现的场景名，不自行补充

### characters

**记录内容**：
- 角色名称
- `identity`：身份定位（职业、社会角色等）
- `personality`：性格概要（核心性格特征）
- `relationship`：与核心角色的关系
- 可选 `birth_year` / `age`：时间跨度大的长篇用 `birth_year`，短篇或时间跨度小可用 `age`
- 可选 `gender`：性别
- 可选 `appearance`：源材料中明确提到的外貌特征（标志性装扮、特殊体貌、伤疤等）
- 可选 `quotes`：角色代表性原文引用，用于后续角色写作时判断性格底色、行为边界、关系动态和说话方式

**要求**：
- 每个有戏份的角色都要记录
- 首次出现时记录，后续有重要变化时更新
- `identity`、`personality`、`relationship` 三个字段各自一句话
- `appearance` 只记录源材料明确提到的特征，源材料没提的外貌不自行补全
- `quotes[].text` 必须逐字来自原文，禁止改写、拼接或概括
- `quotes[].chapter` 必须与 `chapters[].name` 中的章节名一致
- 没有足够代表性的原句时不填 `quotes`

### items

**记录内容**：
- 物品/道具/能力名称
- 一句话描述（特性、用途）

**要求**：
- 只记录对剧情或世界观有重要意义的物品
- 不记录普通日常物品

### important_chapters

**记录内容**：
- 章节名称
- 重要性原因（关键转折、首次出现、名场面等）
- 可选 `quotes`：该章节中支撑重要性的原文原句

**quotes 字段**：
- `text`：原文原句，必须能在源文件中逐字搜索到
- `context`：该原句在章节中的上下文说明
- `function`：该引用的用途，可写人物塑造、剧情高潮、世界观揭示、关系动态等

**选择标准**：
- 只收录人物塑造、剧情高潮、核心世界观揭示的关键原句
- 每章最多 1-2 句
- 不把多个不连续句子拼成一条引用

**示例**：
- chapter: 1-魔女小姐
  reason: 魔女身份首次揭示，核心世界观设定
  quotes:
    - context: 莉莉丝宣告魔药效果，奠定主线身份转变
      function: 世界观揭示
      text: 喝下这瓶魔药，你将成为见习魔女。

**用途**：
- 帮助理解故事结构
- 标记需要重点关注的章节
- 为后续条目规划提供参考

### style_hints

**记录内容**：
- `perspective`：叙事视角（第一人称/第三人称等）
- `tone`：语言风格（口语化/文学化、轻松/严肃等）
- `mood`：情绪基调（温馨/暗黑/搞笑/紧张/浪漫等）

**要求**：
- 从源材料的叙事方式推断，不是评价源材料的好坏
- 只记录源材料客观呈现的风格特征

## 记录原则

- **宁多勿少**：宁可多记录一些，也不要遗漏关键信息
- **具体优于抽象**：记录具体细节，而非抽象概括
- **禁止缩写**：不使用「等」「略」「待补充」等省略标记
- **原文引用必须可验证**：`quotes[].text` 必须逐字复制原文，不写转述句，不补充解释词

## 不需要记录的内容

- 作者的闲聊、求票、求收藏
- 纯环境描写（除非对剧情有重要意义）
- 通用常识（如"人会生病"）
- 与世界观、角色、物品无关的过渡性描写
