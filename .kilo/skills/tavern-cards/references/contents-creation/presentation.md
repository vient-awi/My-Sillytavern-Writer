# 呈现方式

世界书级别的整体条目，定义 AI 呈现内容的方式——叙事基调、文风要求、写作约束等。

## rephrase 区分

### rephrase: false

整体叙事基调、互动要求、文风参考等。

示例：
```
role_play_setting:
  - 以<user>为中心的沉浸模式，一切剧情需围绕<user>展开
  - 严禁赋予<user>上帝视角的认知
  - 误会和修罗场能够增添故事的戏剧性和张力，是恋爱故事的不可或缺的展开。

style:
  name: 校园恋爱喜剧
  key_elements: [恋爱, 轻松搞笑, 青涩]
interaction_rules:
  - 角色不对<user>唯命是从，拥有独立的目标和情感逻辑
  - 当<user>改变关键节点时，推演蝴蝶效应导致的新展开
```

### rephrase: true

强调用户的特殊要求、消除 AI 刻板印象。两三句即可。例如：
- 克苏鲁世界观不希望出现血腥描写。
- 小说同人不希望完全按照原著剧情发展。
- 第三人称叙事，但不希望泄露故事人物未知的信息。

## 编写要点

- 写"如何做"的指令，不写"是什么"的描述
- 负面约束有效，但正面指导更优
- 内容应简洁，通常 200 字以内
