# 区域条目

## 区域条目结构

每个区域条目包含：

- 区域概述：地理特征、气候、氛围
- 地点/场景列表
- 势力/组织分布（如适用）

## part 区分

| part | 内容 | 编写要点 |
|------|------|---------|
| region | 区域概览 | 概览性质，包含地理特征、主要地点、势力分布 |
| scene | 场景详情 | 单个场景的具体描写，信息量足够时从 region 拆出 |
| faction | 势力详情 | 单个势力的具体设定，信息量足够时从 region 拆出 |

## 拆分时机

信息量少时，场景和势力写在区域条目内即可。
信息量足够时拆为独立条目——判断标准：场景或势力的内容独立成段后，是否比留在区域条目里更清晰？是否需要通过条目显隐控制 token？

## 地理速览

当区域数量超过5个且有大量单独的场景、势力条目，可以考虑编写地理速览，列出区域名称、地点/场景、势力/组织的名称。

## 自查清单

- [ ] region 条目包含区域概述、地点/场景列表、势力分布（如适用）
- [ ] scene/faction 拆分合理：信息量足够时才拆，不够时留在 region 内
- [ ] part 取值正确（region / scene / faction），没有混用
- [ ] 通用自查：按 `references/rules.md` 通用自查
