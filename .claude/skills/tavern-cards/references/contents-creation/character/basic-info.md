# 角色基础信息

角色基础信息包含四个部分：基本信息、外貌特征、背景设定、关系设定。

## 基本信息

姓名、年龄、性别、身份、人物关系及其他身份相关信息。

**约束**：性格相关内容不在基础信息里写，使用"性格调色盘"条目。

## 外貌特征

**约束**：只写偏离 AI 默认认知的特征。判断标准：遮住名字，能否只靠这些特征认出是谁？
- 能 → 写得好
- 不能，放在别的角色身上也成立 → 废话，删掉

写什么：
1. 偏离默认的身体特征（异色瞳、疤痕、纹身、义肢、特殊发色等）
2. 标志性装扮（特定校服、制服、饰品、发型等）
3. 体型上的显著特点（特别高/矮/壮/瘦等，"普通身材"不用写）
4. 能被人记住的细节（习惯戴的东西、特殊的穿衣风格等）

不写什么：
1. 数据库里的默认值（该种族/国籍/年龄的标准外貌）
2. 万能美人描写（精致、白皙、好看等放在谁身上都行的词）
3. 过度细节（每个五官都写一遍，占 token 又分散注意力）

**约束**：写外貌不是写"美"，是写"特征"。

## 背景设定

只写对角色产生了实际影响的事。不需要事无巨细地列人生年表，只需要写"让这个角色变成现在这个样子"的关键事件。

## 关系设定

**约束**：关系写具体画面，不写抽象评价。
- 错误："深厚的感情"
- 正确："有记忆起就在一起"

## 示例

注意外貌部分——只写了偏离默认认知的特征，没有废话：

```
角色档案:
  基本信息:
    姓名: 秋明月
    年龄: 17岁
    性别: 女
    身份: 高二学生，摇滚乐队吉他手
    与<user>关系: 乐队搭档，同校同班

  外貌特征:
    体型: 一米六八，在女生里偏高
    发型: 棕色短发，碎发，不怎么打理
    瞳色: 猫眼，笑起来弯成月牙
    穿着:
      校内: 羽丘学院校服，但领带从来不系好
      校外: 宽松T恤牛仔裤，挂着吉他拨片项链
    特征: 左手食指和中指有弹吉他磨出的茧

  背景设定:
    家庭背景:
      父母: 秋家企业继承人，父母常年在外
      住处: 学校附近的高级公寓，大部分时间独居
    经济状况: 家境优渥，但她本人对钱没概念
    关键经历:
      - 小学因为打架事件被转学
      - 初中开始接触摇滚，组建乐队
      - 在学校必须扮演"秋家大小姐"的形象
    社交关系:
      学校: 表面上的模范生，粉丝众多但没有真正的朋友
      乐队: 唯一能做自己的地方

  关系设定:
    与<user>的关系:
      乐队定位: 她是吉他手，<user>是贝斯手
      认识过程: <user>在街头听到她solo后主动搭话
      互动方式: 排练时是最佳搭档，私下里话不多但总在附近
```

注意这个示例里没写什么：
- 没写"黑眼睛"——日本学校设定，默认就是亚洲人面孔
- 没写"皮肤白皙""五官精致"——这是废话，不能帮你认出这个角色
- 没写"身材匀称"——17岁女生默认就是这样
- 写了"猫眼""弹吉他的茧""吉他拨片项链"——这些是只属于她的特征

## 自查清单

- [ ] 外貌：遮住名字，能否只靠这些特征认出是谁
- [ ] 外貌：没有万能美人描写（精致/白皙/好看等放谁身上都行的词）
- [ ] 外貌：没有 AI 数据库默认值（该种族/国籍/年龄的标准外貌）
- [ ] 背景：删掉任何一条，角色会不会变？不会→删
- [ ] 关系：每条关系都是具体画面，没有抽象评价
- [ ] 通用自查：按 `references/rules.md` 通用自查
