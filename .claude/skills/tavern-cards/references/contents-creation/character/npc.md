# NPC 编写

NPC 是功能性角色，围绕功能设计，不写用不到的内容。

**约束**：NPC 不需要完整人设——不写性格调色盘、三面性、二次解释。

**约束**：每个 NPC 一个独立文件。

## 功能定位

功能定位决定设计重点：

| 功能类型 | 设计重点 |
|---------|---------|
| 推动剧情 | 关键行为和决策点 |
| 提供信息 | 专业领域和知识范围 |
| 制造冲突 | 对立动机和行事方式 |
| 日常互动 | 生活习惯和互动场景 |

一个 NPC 可以同时承担多种功能。

## 基础信息

姓名、年龄、性别、身份。

## 外貌特征

**约束**：只写 1-2 个最显眼的关键特征，一句话概括整体印象。遵循 `references/contents-creation/character/basic-info.md` 的外貌原则——只写偏离默认认知的特征，不写万能美人描写。

## 性格核心

**约束**：只提炼 2-3 个核心关键词，不展开。

**约束**：性格写具体行为，不写标签。
- "性格温柔" → 删
- "每次见面都会问要不要帮忙带早餐" → 留

## 与<user>的关系

与<user>的关系是 NPC 最重要的部分——为什么互动、怎么互动。

**约束**：关系写具体画面，不写抽象评价。
- "对<user>很关心" → 删
- "每次<user>路过早餐摊都会多给一个包子" → 留

## 语言特征

说话风格、口头禅（如有）、5-10 句典型对话。

**约束**（语料纯净度）：语料只保留纯对话，删除混入的动作、表情、心理描写。

## 格式示例

```
基本信息:
  姓名: 王静
  年龄: 35岁
  性别: 女
  身份: 高中班主任，教数学

外貌特征:
   整体印象: 一米六五左右，身材微胖
   关键特征: 总皱着眉，眼神严厉
   穿着风格: 深色职业装，平底鞋

性格核心:
  核心特质: 每句话都像在下命令、从不跟学生闲聊、笑的时候学生反而更紧张
  行为模式:
    - 上课时在教室里来回走动盯着学生
    - 发现问题立刻当众指出
    - 经常叫学生到办公室单独谈话

与<user>的关系:
  关系: 班主任
  态度: 每次月考后都会把<user>叫到办公室，把成绩单拍在桌上问怎么回事
  互动方式: 频繁找<user>谈话、批评、施加学习压力

语言特征:
  说话风格: 语气严肃，直接，不绕弯子
  口头禅: "你看看你"、"我跟你说多少次了"
  参考语料:
    - "<user>，来我办公室一趟"
    - "你看看你这次考试，又退步了"
    - "我跟你说多少次了，上课要认真听讲"
    - "你这样下去怎么行"
    - "不要找借口，我只看结果"
    - "回去把这张卷子重做一遍"
```

## 示例对比

**功能性 NPC**（推动剧情）：设计重点是施压行为和说教式对话，推动<user>的学习压力线。

**氛围型 NPC**（日常互动）：设计重点是慵懒的行为模式和简短的对话，提供日常互动的真实感。

两者都没有完整背景故事、没有内心世界、没有性格调色盘——功能够用就好。
