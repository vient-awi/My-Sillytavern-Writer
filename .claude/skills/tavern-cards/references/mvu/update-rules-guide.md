# MVU 变量更新规则（变量更新规则.yaml）

完整的规则编写规范见 `references/mvu/update-rules.yaml`。

## 编写原则

- **自明变量省略**：变量名已说明如何更新时，不写 `check` 规则
- **同类合并**：使用 `${键1|键2|...}` 语法合并同类变量
- **特殊前缀跳过**：`_` / `$` 前缀的变量不写更新规则
- **z.record 区分**：正确区分变量路径和 `type` 字段中的键名
- **type 一致性**：`type` 字段需与 `schema.ts` 中的定义一致

编写前读取 `schema.ts` 提取所有变量定义，确认哪些变量需要较复杂的更新条件、是否有依赖关系、是否有特殊触发条件（如好感度达到阈值触发事件、特定地点才能获得的物品等）。

注册为 `[mvu_update]变量更新规则`。

## 字段说明

变量更新规则中的每个变量路径下可使用以下字段：

| 字段 | 用途 | 示例 |
|------|------|------|
| `type` | 变量支持的类型（`string` 可省略） | `number`、`'未领取'\|'进行中'\|'已完成'` |
| `range` | 数值变量的取值范围 | `0~100` |
| `format` | 变量必须满足的格式 | `YYYY年MM月DD日 星期X HH:MM` |
| `check` | AI 更新变量时应考虑的因素（列表） | 见下方详细说明 |
| `value` | 对变量值的简短描述（用于语义说明） | `主角对某个变量内容的即时感受` |

### `type` — 告诉 AI 变量接受什么值

schema.ts 已规定了类型约束，但 AI 看不到 schema。在此声明让 AI 总是更新为可接受的值：

```yaml
白娅.依存度:
  type: number
任务进度:
  type: '未领取'|'进行中'|'已完成'
```

### `range` — 告诉 AI 数值范围

比起 AI 更新出界值再被 schema 修复，直接告诉 AI 范围更可靠：

```yaml
白娅.依存度:
  type: number
  range: 0~100
```

可附带 `category` 子字段对范围分段释义：

```yaml
主角.能力面板.${力量|敏捷|体质|感知|意志|魅力}.数值:
  type: number
  range: 0~100
  category:
    20~40: 普通人
    40~70: 冒险者常驻
```

### `format` — 告诉 AI 格式要求

不建议在 schema.ts 中用 zod 规定复杂格式（容易导致 AI 更新失败），在此说明即可：

```yaml
世界.当前时间:
  format: ${YYYY/MM/DD}-${HH:MM}
```

### `check` — 告诉 AI 何时更新、更新成什么

用自然语言列表描述更新条件和规则，这是变量更新规则的核心：

```yaml
白娅.依存度:
  check:
    - 根据白娅对<user>行为的感知和反应调整 ±(3~6)

主角.能力面板.${力量|敏捷|体质|感知|意志|魅力}.数值:
  check:
    - 训练、战斗、重伤、系统奖励等显著事件才调整
    - 单次变化不超过 ±10，除非剧情有明确强化/削弱
```

自明变量（变量名已说明如何更新的）不需要写 `check`。

## 示例

```yaml
---
变量更新规则:
  世界:
    当前时间:
      format: ${YYYY/MM/DD}-${HH:MM}
      check:
        - 每次事件推进、休息或旅行后更新，保持时间流逝合理
        - 若场景跳转跨度较大，应说明跳跃原因
  主角:
    能力面板.${力量|敏捷|体质|感知|意志|魅力}.数值:
      type: number
      range: 0~100
      category:
        20~40: 普通人
        40~70: 冒险者常驻
      check:
        - 训练、战斗、重伤、系统奖励等显著事件才调整
        - 单次变化不超过 ±10，除非剧情有明确强化/削弱
    装备栏.${部位}:
      type: |-
        {
          装备: string; // 装备名称 + 状态; 若未装备，使用"空置"或"无"
          主角评价: string;
        }
      check:
        - 穿戴、损毁、替换装备时更新装备描述
  任务列表:
    type: |-
      {
        [任务名: string]: {
          类型: '主线' | '支线' | '每日' | '临危受命' ;
          说明: string; # 面向主角的任务背景或细则
          目标: string; # 明确可执行的目标描述，可包含步骤
          奖励: string;
          惩罚: string; # 失败后触发的负面效果
        }
      }
    check:
      - 避免一次性添加超过3个主线任务，保持焦点
      - 日常任务完成后可重置但需记录冷却
  ${变量}.主角评价:
    value: 主角对某个变量内容的即时感受
    check:
      - 在对应变量值发生变化或遭遇相关事件后可更新，其他情况不应更新
      - 语言应保持第一人称/贴近主角口吻
      - 主角的评价并不会被主角本人看到，也不会在剧情中出现
```
