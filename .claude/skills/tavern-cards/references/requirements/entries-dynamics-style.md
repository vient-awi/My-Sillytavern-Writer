# 条目、MVU 与风格规划

世界与角色信息收集完成后，规划条目、动态变量、EJS、写作风格与开场白。

## 条目规划

根据已收集的信息列出所有条目：

1. 列出条目名称、类型、文件路径
2. 标注 EJS 需求（使用 EJS 时）：
   - 逐条目判定 complexity（条目显隐 / 段落控制 / 动态文本），判定标准和语法详见 `references/ejs/guide.md`
   - 根据要求判断 condition（条件表达式概念，如"当前地点位于华东区时激活"）
   - 一次确认模式：进一步推导 EJS预处理 变量映射
   - 粗略规划模式：EJS预处理 留空，创作阶段推导
3. EJS 信息写入 ejs 段

**entries 是项目级事实来源**，数组顺序即默认创作顺序。

### 条目规划指导

列条目时检查以下几点：

- **rephrase（二次解释）必要时使用**：rephrase 是对同类型条目的二次解释，用于纠正 AI 可能产生的刻板印象或误解。例如角色性格调色盘写了"叛逆、不拘小节"，AI 可能理解为"不良少女"，rephrase 条目澄清"叛逆不是坏学生，是不盲从权威"。简单条目（基础信息、地理速览等）不需要 rephrase。
- **文件夹粒度控制**：同一文件夹下条目超过 5 个时，考虑增加子文件夹分组。例如角色子条目按角色名建子文件夹，地理条目按区域建子文件夹。
- **条目粒度原则**：一个条目只承载一类信息。如果某类信息在特定条件下有截然不同的表现（如三面性），拆分为独立条目而非塞入一个条目加条件分支。
- **拆分/合并判断**：信息少时使用单一世界观条目；有时间线时添加 timeline 条目；有多区域时逐区域展开；信息量足够的场景、势力、区域细节拆为独立条目。
- **阶段指导判断**：跑团、无限流、副本探索、经营循环、任务制玩法等有固定流程时，规划 `阶段指导` 条目；此类型必须使用 EJS 控制，根据 MVU 阶段变量渲染当前阶段的主持目标、互动方式和推进边界。
- **命名可读性**：条目名称应能直接看出内容和用途，避免过于抽象的缩写或编号。entryManifest 的 key、文件名、创作规划 entries 的 name 三者保持一致。
- **scope 最小化**：默认 scope 为 specific（仅相关时激活），只在角色速览、地理速览等需要全局常驻时使用 catalog。

## MVU 变量结构确认

使用 MVU 时，所追踪的变量结构在需求对齐阶段确定，写入 `创作规划.yaml` 的 `mvu` 段落。

确认以下内容：

1. **需要追踪哪些内容？**
   - 核心角色：每个角色需要追踪什么属性？（好感度、位置、状态、装备等）
   - 系统变量：时间、日期、场景、剧情进度等
   - NPC：是否需要批量追踪 NPC 状态？
   - 如果用户没有清晰想法，先根据角色卡主题提出 2-4 个可选追踪方案，不直接替用户决定。建议应具体到变量名称、用途和取值范围：
     - **通用基础状态**：日期、时间段、当前位置。几乎所有 MVU 项目都可使用，用于维持场景连续性、角色出入场和地点相关条目显隐
     - **恋爱/关系推进**：好感度、信任度、亲密阶段、当前关系状态、近期关键事件
     - **校园/日常群像**：课程/社团安排、角色当前行动、角色间近期互动、已发生事件
     - **同人小说/长篇叙事**：当前章节、剧情阶段、原作事件改变量、主线分歧、已触发名场面
     - **冒险/跑团/副本**：当前任务目标、已发现线索、持有道具、危险度、NPC 生死/立场
     - **经营/养成**：资金、资源库存、设施等级、员工状态、客流/声望、待办任务
     - **悬疑/调查**：已知线索、嫌疑人状态、证据链、时间线节点、玩家已触发事件
     - **战斗/异能**：体力/伤势、能力冷却、装备、阵营关系、威胁等级
     - **心理/黑暗向**：压力值、依赖度、警戒度、秘密暴露程度、关系裂痕
   - 每个方案都说明“追踪它会影响什么”：例如 EJS 条目显隐、阶段指导、角色反应、开场白续接或变量更新规则。
2. **取值范围和格式约束**
   - 数值是否有范围？（0~100 等）
   - 文本是否需要使用可选值集合？（`导入｜准备｜战斗｜结算` 等）为了保证 AI 演绎的自由度，仅当用户要求或EJS控制需要才使用可选值。
   - 哪些对象可以动态增删键？（物品栏、成就、技能等）
3. **特殊更新规则**
   - 是否有连锁触发、阶段跃升、自动增减等复杂条件？
   - 简单的递增/递减无需记录，创作阶段按变量性质处理

确认后整理变量结构大纲并写入 `mvu.structure`（一次确认模式下同时写入 `mvu.variables`）：

```
顶层结构：
├── 系统变量
│   ├── 日期：格式
│   ├── 时间：...
│   └── ...
├── 角色1
│   ├── 基础属性
│   │   ├── 好感度：...
│   │   └── ...
│   └── ...
└── 角色2
   └── ...
```

此结构会在条目创作完成后直接用于编写 schema.ts（见 `references/mvu/guide.md`）。

## 写作风格

- 叙事视角（第一人称/第三人称）
- 语言风格（口语化/文学化、轻松/严肃）
- 情绪基调（温馨/暗黑/搞笑/紧张/浪漫）
- 风格参考（可选）：是否有想参考的作家或作品？

## 开场白（角色卡时）

- **开场情节**：故事开始时的情境——角色们处于什么状态？发生了什么事？
- **格式偏好**：叙事式还是大纲式？
  - 一次确认模式：同时收集 scene 和 opening_situation
  - 粗略规划模式：只收集 format
- 叙事式开场白字数范围：如 300~500
