# 条目类型说明

typeLists 配置前，先理解每种条目类型。以下为所有可用类型：

| 类型 | 定义 | 承载信息 | 何时使用 | part | 边界说明 |
|------|------|---------|---------|------|---------|
| **世界观** | 世界观基础设定 | 世界规则、物理法则、社会结构、历史背景等宏观框架 | 任何项目必写 | 无 | 不含具体区域/角色信息；只写会让 AI 出错的差异信息 |
| **扮演准则** | AI 扮演行为约束 | 叙述风格、禁止行为、敏感内容规避、语气要求等 | 用户对 AI 行为有特殊要求时 | 无 | 不是世界观——世界观写"是什么"，扮演准则写"怎么演"；不是角色专属，是世界书级全局约束 |
| **地理** | 空间信息 | 区域概览、场景详情、势力分布等 | 项目涉及多个区域时 | `region`（区域概览）、`scene`（场景详情）、`faction`（势力详情） | 区域概览写宏观印象，场景详情写具体画面；势力涉及多个场景时独立条目 |
| **时间线** | 时间维度信息 | 历史事件、当前剧情进度 | 有明确时间线或需要追踪剧情进度时 | `history`（历史事件）、`plot`（当前剧情） | 不是世界观——世界观写规则，时间线写事件序列 |
| **角色** | 核心角色人设 | 基础档案、性格、三面性等 | 角色扮演类、日常互动类必写 | `basic`、`personality`、`tri_faceted`；rephrase 条目使用 `personality` | 每个核心角色至少写 basic + personality；NPC 不用此类型，用 NPC 类型 |
| **NPC** | 非核心角色 | 功能性角色的简要信息 | 有配角/NPC 需要定义时 | 无 | 功能定位简要即可，不写 personality/tri_faceted；信息量达到角色级别时升级为角色类型 |
| **事件** | 特定事件/场景 | 一次性或条件触发的事件描述 | 有不属于地理/时间线的特殊事件时 | 无 | 不是时间线——时间线写事件序列，事件写单次触发的内容 |
| **阶段指导** | 阶段化玩法/剧情主持策略 | 不同阶段的目标、叙事重点、互动方式、推进边界 | 跑团、无限流、副本、经营循环、任务制玩法等有固定流程时 | 无 | 必须使用 EJS 控制；通常合并为 `scope: catalog` 的总指导条目，通过段落控制渲染当前阶段；不是时间线——时间线写事件顺序，阶段指导写每个阶段怎么主持；不是事件——事件写单次触发内容，阶段指导写可复用流程规则 |
| **EJS预处理** | EJS 模板的变量预处理 | `@@generate_before` 装饰的 define() 语句，为其他条目提供 EJS 变量 | 使用 EJS 时必写 | 无 | 仅在使用 EJS 时存在；内容为 EJS 代码，不是常规世界书内容 |
| **MVU** | MVU 变量系统 | 初始变量、变量列表、更新规则、输出格式 | 使用 MVU 时必写 | `initvar`、`variable_list`、`update_rules`、`output_format` | 仅在使用 MVU 时存在；变量相关条目由 MVU 收尾流程自动注册 |

## 物品归属（材料转化）

`故事大纲.yaml` 中收集的 `items`（物品、道具、能力、技能）没有独立条目类型，按以下规则归入现有类型：

| 物品特征 | 归属类型 | 示例 |
|----------|----------|------|
| 世界规则级物品（影响整体设定） | **世界观** | 魔女学派体系、收容物评判标准 |
| 区域/势力专属物品 | **地理.faction** | 修仙协会的执法令、暗月教的禁忌法典 |
| 角色专属物品（与角色强绑定） | **角色.basic** | 苏云的灵纹疤痕、莉莉丝的魔杖 |

**判断原则**：
- 物品是否影响世界观规则？→ 世界观
- 物品是否属于特定势力？→ 地理.faction
- 物品是否与角色绑定（角色标志性物品）？→ 角色.basic
- 以上都不是 → 按物品出现时机归入 时间线.plot
