# 写作规则

所有世界书条目编写必须遵守的规则。本文档是唯一的写作规则来源，其他文档不再重复。

## 格式规范

- 使用 YAML 中文格式
- 缩进用 2 空格
- 用 xml 标签包裹条目内容（如需要）
- 数据库格式优先，不是散文：用列表和键值对，不用段落
- 语法冲突规避 / 多条目合并：参见 `references/conventions.md`
- **全文简体中文**：所有条目内容、变量名、文件名使用简体中文，不使用繁体字或日文汉字（如"桜"→"樱"、"宮"→"宫"、"靜"→"静"、"繪"→"绘"）。转化阶段需检查源材料，在转为条目内容前统一替换为简体
- **禁止占位符**：条目内容中不出现"某城市""某学校""某组织"等含糊表述。所有关键具体名称（地名、校名、店名、角色名等）必须明确写入。不确定时暂停创作并询问用户，不得自行捏造或用模糊词替代

## 角色写作要点

针对角色相关条目（角色基础信息、性格调色盘、三面性、二次解释等）的正面写作指引。

### 外貌只写特征

只写偏离 AI 数据库默认认知的部分。判断标准：遮住名字能否认出是谁。

- ✓ "左眼角下有一颗泪痣"
- ✗ "她有一头长发，眼睛很大"（这是默认认知）

### 背景只写关键事件

删掉这条背景，角色会不会变？不会则删。

- ✓ "七岁那年父亲失踪，此后再未提起"
- ✗ 大段成长经历流水账

### 关系写具体画面

用具体画面替代抽象标签。

- ✓ "有记忆起就在一起"
- ✗ "她们感情深厚"

### 行为展现性格

用具体行为展现性格，不贴标签。

- ✓ "遇到受伤的小动物会带回家照顾"
- ✗ "性格温柔善良"

### 一句一意

写完一个态度判断就直接停，不补述同一件事。

- ✓ "长辈安排下来我会照着做"
- ✗ "长辈安排下来我会照着做。这件事我从小就习惯了"（第二句是同义补述）

## 世界观压缩

针对世界观相关条目（世界观概览、区域条目、时间线条目等）的专项指引。

### 压缩原则

- 连接词用冒号/逗号替代
- 不写主观评价
- 不写 AI 已知信息
- 只写让 AI 会出错的差异信息

### 每句话过四问

1. 删了这句 AI 会错吗？不会 → 删
2. 是信息还是装饰？装饰 → 改写或删
3. 列表能替代吗？能 → 改列表
4. 不看原文能理解吗？不能 → 补关键信息
