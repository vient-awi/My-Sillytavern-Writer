# 周回指南

## 角色 CG 为什么能跨周回保存

角色 CG 解锁记录使用的是酒馆助手的角色卡变量，而不是 MVU 变量。

核心变量名：

```ts
性斗学园CG解锁记录
```

核心作用域：

```ts
{ type: 'character' }
```

`{ type: 'character' }` 表示数据绑定在当前角色卡上。它不绑定某个聊天文件，也不绑定某个消息楼层。因此，只要仍然使用同一张角色卡，重新开聊天、切换聊天、进入新周回时，角色 CG 解锁记录仍然可以被读取。

## 和 MVU 的区别

MVU 变量框架主要作用在消息楼层变量上，常见读取方式是：

```ts
Mvu.getMvuData({ type: 'message', message_id: ... })
```

这类数据通常存放在 `stat_data` 中，适合保存当前剧情、战斗状态、好感度、任务进度等会随聊天推进变化的内容。

但角色 CG 解锁记录属于“收藏型永久记录”。如果放进 MVU，它会和某个聊天或楼层强绑定，换聊天或开新周回时就不适合继续复用。因此性斗学园把 CG 解锁记录独立放进角色卡变量。

## 实际读写位置

实现文件：

- `shared/cgUnlockStore.ts`

读取角色卡变量：

```ts
getVariables({ type: 'character' })
```

写入角色卡变量：

```ts
insertOrAssignVariables(
  { 性斗学园CG解锁记录: unlockMap },
  { type: 'character' },
)
```

数据结构大致是：

```json
{
  "角色名A": ["cg事件id-0", "cg事件id-1"],
  "角色名B": ["cg事件id-0"]
}
```

其中 CG key 的格式是：

```ts
`${cgId}-${imageIndex}`
```

## 解锁流程

战斗结束后，战斗界面会根据敌人、玩家性别、胜负结果选择一张 CG。

流程如下：

1. `战斗界面/app.vue` 调用 `selectCombatCG()`。
2. `战斗界面/combatConclusion.ts` 根据战斗结果选择 CG。
3. 代码生成 `cgKey`。
4. 调用 `unlockCharacterCGKey(enemyName, cgKey)`。
5. `shared/cgUnlockStore.ts` 把该角色的 CG key 写入 `性斗学园CG解锁记录`。
6. CG 图鉴页读取角色卡变量，并用 key 判断图片是否已解锁。

## 图鉴读取流程

图鉴页实现文件：

- `性斗学园脚本/components/pages/CGPage.vue`

图鉴页加载时会调用：

```ts
getAllUnlockedCGKeysByCharacter()
```

然后转换成“角色名 -> Set<CG key>”的形式。显示每张图时，会用：

```ts
isCGImageUnlocked(characterName, cgId, imageIndex)
```

判断是否已解锁。

图鉴页还会监听两个事件：

```ts
cg-unlocked
cg-unlock-records-updated
```

这些事件用于在战斗结算、旧数据迁移、满好感自动解锁后刷新图鉴显示。

## 旧数据迁移

旧版本可能存在两类 CG 记录：

- 本地存储里的旧 CG key。
- MVU 关系系统里的 `已解锁CG` 字段。

新版本会尝试把这些旧记录迁移进角色卡变量 `性斗学园CG解锁记录`。

迁移入口在：

- `shared/cgUnlockStore.ts`
- `性斗学园脚本/index.ts`

迁移完成后，不应该再手动向 MVU 里新增 `已解锁CG` 字段。

## 满好感自动解锁

主脚本会检查 MVU 里的 `关系系统`。

当某个角色的 `好感度 >= 100` 时，会调用：

```ts
unlockMaxFavorCharacterCGsFromMvuData(mvuData)
```

该函数会把这个角色配置中所有 CG key 写入角色卡变量，实现满好感自动解锁全部 CG。

## 周回设计结论

角色 CG 的跨周回存储依赖酒馆助手角色卡变量：

```ts
{ type: 'character' }
```

MVU 负责当前周回、当前聊天、当前楼层里的剧情状态；角色卡变量负责跟随角色卡存在的长期记录。

因此，本项目里的角色 CG 解锁记录可以理解为：

```txt
当前周回数据：MVU / stat_data
跨周回收藏：角色卡变量 / 性斗学园CG解锁记录
```

如果以后还要做跨周回永久记录，例如成就、已看过的特殊剧情、永久解锁的功能，也应该优先考虑使用角色卡变量，而不是写进 MVU。

## 注意事项

- 角色卡变量只跟随当前角色卡，不是全局共享。换成另一张角色卡时，不会自动共享这份 CG 记录。
- 如果角色名无法匹配 CG 配置，解锁记录可能无法写入对应角色。
- 如果酒馆助手的 `insertOrAssignVariables` 不可用，代码会尝试回退到旧本地存储，但这不是新版本主路径。
- 如果满好感未自动解锁，优先检查 `关系系统` 中角色名是否规范，以及好感度是否达到 `100`。
