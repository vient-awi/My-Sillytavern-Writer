# 性斗学园项目总解析

本文档基于 `F:\酒馆模板\性斗学园` 当前源码整理，目标是说明这个项目包含哪些功能、功能分别依赖哪些文件实现、数据如何流转，以及运行时和酒馆环境的关系。

## 1. 项目定位

`性斗学园` 不是单一页面，而是一套完整的酒馆助手前端扩展。它由多个独立 Vue 入口、共享 MVU 数据层、回合制战斗系统、小手机管理面板、后街聊天系统、世界书同步和自动更新脚本组成。

核心结构可以概括为：

```text
酒馆消息 / 世界书 / MVU
        ↓
开局 UI 初始化角色
        ↓
正文 UI / 选项 UI / 变量更新 UI / 并行事件 UI 解析模型输出标签
        ↓
选项 UI 触发 <fight> 战斗楼层
        ↓
战斗界面读取 MVU + 敌人库 + 技能库，进行回合制结算
        ↓
战斗结果写回 MVU、奖励、CG 解锁
        ↓
主脚本挂载小手机，持续修正等级、段位、恢复、世界书和后街注入
        ↓
小手机管理资料、背包、技能、商店、关系、地图、CG、后街聊天
```

项目目录：

```text
性斗学园/

```

当前根构建配置在 `F:\酒馆模板\webpack.config.ts`，主要通过 `src/**/index.{ts,tsx,js,jsx}` 扫描入口。而 `性斗学园` 目录不在 `src` 下，所以它不是被当前 webpack glob 直接扫描到的普通入口。实际构建或发布很可能依赖额外复制、符号链接、同步脚本或外部配置。

各 UI 子目录里的 `index.ts` 基本都是 Vue 挂载入口，真正逻辑在对应 `app.vue` 或组件文件中。

## 3. 数据与存储层

### 3.1 MVU Schema

核心变量定义在：

```text
性斗学园/变量结构/schema.ts
性斗学园/变量结构/schema.json
性斗学园/变量结构/README.md
```

`schema.ts` 定义了角色卡和玩法运行依赖的主状态结构，主要包括：

| 模块 | 作用 |
|---|---|
| `角色基础` | 姓名、等级、经验、段位、难度、性别等 |
| `核心状态` | 耐力、最大耐力、快感、最大快感、属性点、技能点、堕落度、潜力等 |
| `基础属性` | 魅力、幸运、闪避率、暴击率等基础值 |
| `临时状态` | 战斗和道具产生的短期 Buff/Debuff |
| `永久状态` | 体质、永久加成、性癖奖励等长期状态 |
| `性斗系统` | 当前对手、胜负规则、高潮上限等 |
| `关系系统` | 在场人物、角色好感度、屈服度、关系类型 |
| `任务系统` | 主线任务、支线任务、完成记录 |
| `物品系统` | 学园金币、背包、装备栏 |
| `位置系统` | 地点名称、坐标、区域 |
| `时间系统` | 日期、星期、时段 |
| `势力声望` | 各势力声望数值 |
| `技能系统` | 主动技能、天赋 |

### 3.2 MVU 读写封装

核心文件：

```text
性斗学园/shared/mvuStore.ts
```

职责：

- 等待 MVU 初始化。
- 读取最新楼层 MVU。
- 替换最新 MVU。
- 按路径局部更新 `stat_data`。
- 清理废弃路径，例如旧版把魅力、幸运、闪避率、暴击率放在 `核心状态` 下，新结构要求这些基础属性在 `基础属性` 下。

主要函数：

| 函数 | 作用 |
|---|---|
| `waitForMvu` | 等待 MVU 框架可用 |
| `getLatestMvuData` | 读取最新 MVU 数据 |
| `getLatestStatData` | 读取最新 `stat_data` |
| `replaceLatestMvuData` | 替换最新 MVU |
| `updateLatestMvuData` | 基于回调修改 MVU |
| `updateLatestStatData` | 用路径批量更新 `stat_data` |
| `sanitizeMvuData` | 清理禁用或废弃路径 |

### 3.3 派生属性与状态引擎

核心文件：

```text
性斗学园/shared/statSelectors.ts
性斗学园/shared/statusEngine.ts
性斗学园/shared/combatMath.ts
性斗学园/shared/combatLimits.ts
```

职责：

- 读取基础属性。
- 合并装备加成。
- 合并永久状态加成。
- 合并临时状态加成。
- 生成玩家和敌人的战斗快照。
- 标准化高潮次数上限。
- 处理状态持续回合、状态插入、状态移除。

典型链路：

```text
MVU stat_data
  ↓
statSelectors 读取基础属性、装备、状态
  ↓
combatMath 计算性斗力、忍耐力、闪避、暴击等派生值
  ↓
战斗界面 / 手机资料页 / 总览页 使用最终快照
```

### 3.4 本地偏好、图片、CG、玩家预设

核心文件：

```text
性斗学园/shared/localPreferences.ts
性斗学园/shared/indexedImageStore.ts
性斗学园/shared/cgUnlockStore.ts
性斗学园/shared/playerPresetStore.ts
```

职责：

- `localPreferences.ts`：保存当前聊天域下的头像、作弊码、旧 CG 解锁、特殊战斗解锁等。
- `indexedImageStore.ts`：用 IndexedDB 保存图片 Blob，返回内部引用，如头像、后街附件、NovelAI 插图。
- `cgUnlockStore.ts`：CG 解锁记录从旧 MVU 字段迁移到角色变量 `性斗学园CG解锁记录`，并支持满好感自动解锁。
- `playerPresetStore.ts`：保存、读取、重命名、删除玩家开局预设。

### 3.5 用户信息与世界书同步

核心文件：

```text
性斗学园/shared/userWorldbookSync.ts
```

职责：

- 保存当前聊天的用户信息到聊天变量 `性斗学园当前聊天用户信息`。
- 把用户信息写入世界书 `性斗学园` 下的 `user` 条目。
- 优先使用酒馆助手函数，失败时回退到世界书读取/替换或 slash 命令。

## 15. 总结

`性斗学园` 的核心是一个以 MVU 为主状态、以世界书和本地存储为辅助状态的互动角色卡系统。

它的主要功能可以分成八类：

1. 建卡与生活模拟开局。
2. 正文、选项、变量更新、并行事件等楼层美化。
3. 回合制战斗。
4. Boss、驱魔、七罪、阶段机制。
5. 奖励、装备、技能、天赋、商店、转盘。
6. 小手机资料/背包/任务/关系/地图/CG 管理。
7. 后街聊天、群聊、图片、主线注入。
8. 世界书和脚本更新检查。

从工程结构看，战斗系统和后街系统是体量最大、逻辑最复杂的两部分。战斗系统偏玩法规则和数值结算，后街系统偏持久化、提示词拼接、世界书读写和生成 API 编排。小手机则是把这些状态可视化并提供手动管理入口。

本次解析只基于源码静态分析，未运行 build，也未验证酒馆运行时行为。
