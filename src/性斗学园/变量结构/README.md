# 性斗学园 MVU v2 写入规范

本文档是世界书、提示词和脚本写入 MVU 时的唯一口径。当前目标是重置存档结构，不兼容旧存档。

## 核心原则

MVU 只保存需要跨楼层继承的事实数据；所有可由事实数据计算出的值，都不要写入 MVU。

- 保存：玩家基础资料、玩家资源、基础属性、背包、装备栏、技能、天赋、任务、关系、位置、时间、当前对手名称和胜负规则。
- 不保存：当前回合、高潮次数、战斗摘要、实时性斗力、实时忍耐力、加成统计缓存、装备总加成、对手当前耐力、对手当前快感、对手状态、对手技能冷却。
- 对手数据通过 `性斗系统.对手名称` 匹配 `战斗界面/enemyDatabase.ts`，战斗界面进入时重新初始化。
- 自定义头像、已激活作弊码、七宗罪层数等 UI/运行时状态使用脚本本地状态或 localStorage，不进入 MVU。
- 角色 CG 解锁记录保存到角色变量 `性斗学园CG解锁记录`，跟随角色卡，不进入 MVU。

## 可写路径

### 角色与资源

```yaml
角色基础:
  _等级: 1
  _姓名: ''
  经验值: 0
  声望: 0
  _段位: 无段位
  难度: 普通
  性别: 女
核心状态:
  $属性点: 0
  $技能点: 0
  $最大耐力: 100
  $耐力: 100
  $最大快感: 100
  $快感: 0
  堕落度: 0
  _潜力: 5
基础属性:
  _魅力: 10
  _幸运: 10
  _闪避率: 0
  _暴击率: 0
```

`基础属性` 下的 `_魅力`、`_幸运`、`_闪避率`、`_暴击率` 由脚本维护；下划线前缀表示不应由 AI 直接写入。

### 状态

临时状态必须写入 `临时状态.状态列表`。不要写 `临时状态.加成统计`。

```yaml
临时状态:
  状态列表:
    贤者时间:
      剩余回合: 3
      描述: 高潮后的短暂恢复期
      加成:
        基础性斗力成算: -20
        基础忍耐力成算: 10
```

永久状态必须写入 `永久状态.状态列表`。不要写 `永久状态.加成统计`。

```yaml
永久状态:
  状态列表:
    禁欲学者:
      描述: 难以被心理攻势影响
      加成:
        基础忍耐力成算: 20
        暴击率加成: 15
```

### 战斗

```yaml
性斗系统:
  对手名称: 白石响子
  性斗类型: 日常切磋
  胜负规则:
    高潮次数上限: 1
    允许认输: true
```

只允许保存进入战斗所需的配置。当前回合、双方高潮次数、战斗摘要、对手当前耐力、快感、状态、技能冷却、可用技能都属于战斗运行态。

### 物品与技能

装备加成从 `物品系统._装备栏` 实时计算；不要写 `物品系统.装备总加成`。

```yaml
物品系统:
  学园金币: 0
  背包: {}
  _装备栏:
    主装备:
      名称: ''
      等级: C
      描述: ''
      加成属性: {}
技能系统:
  主动技能: {}
  $天赋: {}
```

主动技能写完整结构；当前冷却不写入 MVU。

```yaml
技能系统:
  主动技能:
    指尖挑逗:
      基本信息:
        技能名称: 指尖挑逗
        技能描述: 以手指轻轻挑逗对手。
        基础描述: 以手指轻轻挑逗对手。
        技能等级: 1
        稀有度: C
      冷却与消耗:
        耐力消耗: 10
        冷却回合数: 0
      伤害与效果:
        伤害来源: 性斗力
        系数: 60
        基础命中率: 90
        连击数: 1
        准确率: 100
        暴击修正: 0
        效果列表: {}
```

## 旧字段迁移口径

```text
核心状态._魅力       -> 基础属性._魅力
核心状态._幸运       -> 基础属性._幸运
核心状态._闪避率     -> 基础属性._闪避率
核心状态._暴击率     -> 基础属性._暴击率
核心状态._最大耐力   -> 核心状态.$最大耐力
核心状态._耐力       -> 核心状态.$耐力
核心状态._最大快感   -> 核心状态.$最大快感
核心状态._快感       -> 核心状态.$快感
性斗系统.$可用技能   -> 技能系统.主动技能
性斗系统.$技能冷却   -> 不迁移，战斗运行态
性斗系统.当前回合    -> 不迁移，战斗运行态
性斗系统.高潮次数    -> 不迁移，战斗运行态
性斗系统.战斗摘要    -> 不迁移，战斗结束时直接创建 user 楼层提示词
```

## 派生值读取

代码需要展示数值时，应使用 selector 计算：

- 玩家派生属性与战斗双方快照：`shared/statSelectors.ts`
- 状态加成汇总：`shared/statusEngine.ts`
- 基础公式：`shared/combatMath.ts`
