### 3. 战斗系统

#### 核心机制

- **回合制流程**：玩家行动 → 敌人行动 → 回合结算
- **伤害计算**：
  ```
  基础伤害 = Σ(属性值 × 系数 + 基础值)
  实际伤害 = 基础伤害 × 40 ÷ (忍耐力 + 100)
  最终伤害 = 实际伤害 × Buff修正
  ```
- **闪避判定**：命中率 = 技能命中率 - 目标闪避 + (攻击者幸运/10)
- **暴击判定**：暴击率 = 基础暴击 + (幸运/5) + 技能修正，暴击伤害×1.5

#### 技能系统

- **技能数据库**：集中管理所有技能，支持多种类型
  - 物理技能（指尖挑逗、深吻、衣物破坏等）
  - 精神技能（污言秽语、嘲讽挑衅等）
  - 辅助技能（弱点洞察、喘息等）
  - 控制技能（束缚等）
- **技能冷却**：每回合自动递减，只保存在本次战斗运行态
- **耐力消耗**：使用技能需要消耗对应耐力值

#### Buff/Debuff系统

支持17种状态效果：

- **增益**：攻击提升、防御提升、暴击提升、集中、持续回复等
- **减益**：防御下降、敏感、沉默、束缚、闪避下降等
- **特殊**：羞耻、发情、乏力、持续快感等
- **叠加机制**：部分buff可叠加，有最大层数限制
- **持续时间**：每回合自动递减，到期自动移除

#### 其他机制

- 高潮判定与处理
- 敌人AI（技能预告）
- 胜负判定

## 使用方法

### 自动加载敌人数据

在世界书的MVU变量中，只需设置：

```yaml
性斗系统:
  对手名称: '白石响二' # 设置对手名称
```

战斗界面会自动：

1. 从数据库加载对应角色的完整数据
2. 包括等级、属性、技能等
3. 在战斗运行态初始化对手资源、状态和技能冷却

### 支持的对手名称

可以使用数据库中的任何角色名称，例如：

- `白石响二` - S班教师，父性支配者
- `艾伦海德` - 学生会长，秩序的化身
- `沙恩斯通` - 男权会长，足技王者
- `明日郎` - BF社社长，手技大师
- `加藤鹰子` - 传说中的神之手指
- `阿尔伯特温特` - 神秘的学生会顾问
- ... 等70+角色

### 未找到角色时的处理

如果对手名称在数据库中不存在，系统会使用默认的"风纪委员长"作为后备。MVU v2 不再保存对手属性副本。

### 技能系统使用

#### 为玩家设置技能

在MVU变量中设置：

```yaml
技能系统:
  主动技能:
    指尖挑逗:
      基本信息:
        技能名称: 指尖挑逗
        技能描述: 以手指轻轻挑逗对手。
        基础描述: 以手指轻轻挑逗对手。
        技能等级: 1
        稀有度: C
      冷却与消耗:
        耐力消耗: 10
        冷却回合数: 0
      伤害与效果:
        伤害来源: 性斗力
        系数: 60
        基础命中率: 90
        连击数: 1
        准确率: 100
        暴击修正: 0
        效果列表: {}
```

**基础技能**：

## MVU变量结构

### 玩家数据读取路径

```
核心状态.$最大耐力
核心状态.$耐力
核心状态.$最大快感
核心状态.$快感
核心状态._潜力
基础属性._魅力
基础属性._幸运
基础属性._闪避率
基础属性._暴击率
临时状态.状态列表
永久状态.状态列表
物品系统._装备栏
技能系统.$天赋
性斗系统.胜负规则.高潮次数上限     # 双方共享的高潮次数上限
技能系统.主动技能                  # 玩家可用的技能列表
```

### 敌人数据读取路径

```
性斗系统.对手名称            # 关键：用于从敌人数据库加载角色
```

当前回合、双方高潮次数、战斗摘要、对手耐力、快感、状态、技能冷却与实时属性都只存在于本次战斗运行态；刷新后重新进入战斗。战斗结束时会直接用本次日志创建 user 楼层提示词，不经由 MVU 保存摘要。

## 扩展指南

### 添加新角色

在 `enemyDatabase.ts` 中添加新条目：

```typescript
'新角色名': {
  name: '新角色名',
  level: 60,
  potential: 8.0,
  avatarUrl: 'https://picsum.photos/400/600?random=XXX',
  stats: {
    charm: 70,
    luck: 60,
    evasion: 20,
    crit: 15,
    maxEndurance: 160,
    maxPleasure: 100,
  maxClimaxCount: 3
  },
  skills: [SKILL_性斗攻击, SKILL_极速手技],
  description: '角色描述'
}
```

### 添加新技能

在 `enemyDatabase.ts` 中定义新技能：

```typescript
const SKILL_新技能: Skill = {
  id: 'new_skill_1',
  name: '新技能名',
  description: '技能描述',
  cost: 0,
  type: 'attack',
  cooldown: 2,
  currentCooldown: 0,
  effect: (user, target) => {
    const dmg = Math.floor(user.stats.sexPower * 1.5);
    target.stats.currentPleasure += dmg;
    return createLog(`${user.name} 使用新技能，造成 ${dmg} 点伤害！`, 'enemy', 'damage');
  },
};
```

## 注意事项

1. **MVU依赖**：界面需要MVU变量框架正常运行
2. **命名规范**：对手名称必须与数据库中的完全匹配（区分大小写）
3. **数据同步**：战斗过程中只保存玩家侧事实数据；对手当前耐力、快感、状态和冷却不写入MVU
4. **性能考虑**：70+角色数据已优化加载，不会影响性能

## 开发建议

- 使用 `console.info` 查看加载日志
- 检查浏览器控制台确认角色是否正确加载
- 测试不同角色确保技能平衡性
- 根据实际游戏需求调整属性数值
